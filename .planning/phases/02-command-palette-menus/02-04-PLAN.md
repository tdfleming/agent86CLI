    ```python
    from agent86.tui.commands import COMMANDS  # module-local import ok (already textual-land)
    palette = self.query_one("#palette", OptionList)
    if not text.startswith("/") or " " in text:
        palette.display = False
        return
    matches = [c for c in COMMANDS if c.name.startswith(text)]
    palette.display = bool(matches)
    if matches:
        palette.clear_options()
        palette.add_options(
            Option(f"{c.name}  [dim]{c.description}[/dim]", id=c.name) for c in matches
        )
        palette.highlighted = 0
    ```
    (The `" " in text` guard hides the palette once the user types an argument — that is the typed
    D-11 path, not palette territory.)

    - If Approach B: do NOT add an App-level `enter` binding. Instead, at the TOP of

      before the existing dispatch, so an open palette consumes Enter and a closed palette leaves
      the existing submit behavior untouched.

    Implement:
    - `action_palette_up`/`action_palette_down`: guard on `palette.display`; if visible call
      `palette.action_cursor_up()` / `action_cursor_down()`.
    - `action_palette_dismiss`: if `palette.display`, set `palette.display = False`; else return
      (leave Input's own Escape behavior alone).
    - `_select_palette(self)` (and, for Approach A, `action_palette_select` delegates to it when
      visible): read the highlighted option's id (the command name); hide the palette; clear the
      `#prompt` Input value; then `self._run_or_chain(find_command(name))`.
    - `_run_or_chain(self, entry)`:
      ```python
      if entry.needs_choice == "mode":
          self.push_screen(ModePickerModal(self.repl.harness.gate.mode.value), self._on_mode_picked)
      elif entry.needs_choice == "model":
          choices = model_choices(self.repl.cfg)
          if not choices:
              prompt = self.query_one("#prompt", Input); prompt.value = "/model "; prompt.focus(); return
          self.push_screen(ModelPickerModal(choices), self._on_model_picked)
      else:
          self._dispatch_line(entry.name)
      ```
    - `_on_mode_picked(self, value)`: `if value is not None: self._dispatch_line(f"/mode {value}")`.
    - `_on_model_picked(self, value)`: `if value is not None: self._dispatch_line(f"/model {value}")`.
    - `_dispatch_line(self, line: str)`: extract the existing body of `on_input_submitted` (the
      log-write + `handle_command` + exit/turn/handled branch + footer refresh) into this shared
      helper so BOTH typed input and picker callbacks run the identical execution path. Have
      `on_input_submitted` (for the closed-palette / typed case) delegate to `_dispatch_line(line)`.

    OptionList selects too).

    synthesize a command line and go through `handle_command` (Pattern 5 / anti-pattern guard).
    Do NOT add any import to cli.py — all of this stays inside tui/.
