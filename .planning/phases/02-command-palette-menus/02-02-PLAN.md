  <action>
    Create `tests/tui/test_palette_keys.py`. Build a minimal spike App subclass (a subclass of
    `Agent86App`, or a small standalone `App` with a single `Input(id="prompt")`) that adds:

    ```python
    BINDINGS = [Binding("enter", "palette_select", show=False, priority=True)]
    palette_open = False       # simulate the future palette-open guard
    submitted: list[str] = []

    def action_palette_select(self) -> None:
        if not self.palette_open:
            return             # no-op when palette closed — the case under test

    def on_input_submitted(self, event) -> None:
        self.submitted.append(event.value)
    ```
    (If subclassing Agent86App, use its real `on_input_submitted` and assert observable side
    effects instead — either approach is fine; the point is to detect whether Input.Submitted fires.)

    Write `test_priority_enter_binding_falls_through_when_palette_closed(tmp_path)`: run under
    `async with app.run_test() as pilot`, set `#prompt` value to "hello world" (plain text, no
    leading '/'), `await pilot.press("enter")`, `await pilot.pause()`. Assert the submit path ran
    (`app.submitted == ["hello world"]`, or for the Agent86App variant assert a turn started /
    status.working became True).

    IMPORTANT — this test records the answer, it must not be a silent xfail:
    - If the assertion PASSES, the permanent priority-`enter`-binding approach is safe → record
      "Approach A" in the SUMMARY.
    - If Textual consumes the event so Input.Submitted never fires, invert the test into the
      documented fallback: assert that WITHOUT the App-level `enter` binding (or with a
      dynamically added/removed binding) submit works, and record "Approach B" (track palette-open
      state and handle Enter inside `on_input_submitted` / dynamically bind-unbind `enter`) in the
      SUMMARY. Either way the committed test must be green and its name/docstring must state which
      approach was confirmed.

    Also add `test_up_down_are_free_keys_on_input(tmp_path)`: assert that pressing "up"/"down" on
    the focused `#prompt` Input does not raise and does not submit (confirms Input has no native
    up/down binding to conflict with the palette's priority up/down bindings).
