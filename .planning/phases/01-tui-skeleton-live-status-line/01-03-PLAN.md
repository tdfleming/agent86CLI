<objective>
Port the existing REPL slash-command dispatch to a Textual-friendly adapter that RETURNS renderables
(and signals actions) instead of printing to a Rich `Console`, so the app can write command output
into the transcript. Reuses the proven `_Repl` object (harness/state/status + `_refresh_status`) and
the pure helpers (`cycle_mode`, `parse_mode`, `harness.set_model`) — `ui/repl.py` and the plain loop
stay untouched.

Purpose: TUI-01 requires "existing slash-command behavior reaches parity" in the TUI. `_Repl.dispatch`
prints directly to stdout, which won't render inside a Textual app; this adapter captures the same
information as return values.
Output: `src/agent86/tui/commands.py` + a headless unit test mirroring `tests/integration/test_repl.py`.
</objective>

  <action>
    Create `src/agent86/tui/commands.py`. Do NOT print to stdout; return values the app writes to a
    RichLog. Define a result type and the dispatcher:
    ```python
    from __future__ import annotations
    from dataclasses import dataclass
    from typing import Any
    from agent86.guardrails.policy import cycle_mode, parse_mode

    @dataclass
    class CommandResult:
        action: str                 # "handled" | "turn" | "exit" | "noop"
        render: Any | None = None   # str or a Rich renderable (Table/Text) to write to the transcript

    def handle_command(repl, line: str) -> CommandResult: ...
    ```
    Reproduce `_Repl.dispatch`'s branch behavior, but build renderables instead of printing:
    - empty line -> `CommandResult("noop")`
    - `/exit` or `/quit` -> `CommandResult("exit")`
    - `/help` -> build the same rich `Table` as `_print_help` (rows for each command) and return it as `render`
    - `/config` -> return `repl.cfg.model_dump_json(indent=2)` (a str) as render
    - `/models` -> build a rich `Table` of providers + a roles table (mirror `_list_models`) and return it
    - `/tools` -> `CommandResult("handled", "tools: " + ", ".join(repl.harness.registry.names()))`
    - `/skills` -> lines for each `repl.harness.skills` value ("<name> - <description>"), or "no skills discovered"
    - `/memory` -> counts summary if `repl.harness.memory` else "memory is disabled"
    - `/cost` -> steps/in/out/cost summary from `repl.state.usage` and `repl.state.step_count`
    - `/clear` -> `repl.state = repl.harness.new_session()`; return `CommandResult("handled", "conversation cleared")`
    - `/mode` / `/mode <arg>` -> if no arg call `repl._cycle_approval()`; else `parse_mode(arg)` and set
      `repl.harness.gate.mode` + `repl.status.approval` (return a message for unknown mode); return
      `CommandResult("handled", f"approval mode: {repl.harness.gate.mode.value}")`
    - `/model` / `/model <arg>` -> if no arg return current-model + usage hint; else try
      `repl.harness.set_model(arg)` catching `(ProviderError, ValueError)` (import ProviderError from
      `agent86.cognitive.base`) — on error return the error text; on success call
      `repl._refresh_status()` and return `f"model: {new.name}:{new.model}"`
    - any other `/...` -> `CommandResult("handled", f"unknown command {line}")`
    - otherwise -> `CommandResult("turn")`
    Add a helper `startup_notes(repl) -> list[str]` returning the same note strings as
    `_Repl.print_notes` (memory/mcp/sandbox/skills/session id) so the app can render them into the
    transcript on mount. Add `__all__ = ["CommandResult", "handle_command", "startup_notes"]`.
  </action>
