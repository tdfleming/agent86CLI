    - An `ApprovalRequest` message pushes `ApprovalModal`; resolving it sets `box["ok"]` and calls

    1. `class Agent86App(App):`
       - `BINDINGS = [Binding("shift+tab", "cycle_mode", "cycle approval mode")]`
         (Textual uses the literal "shift+tab"; `cfg.ui.mode_cycle_key` is prompt_toolkit's "s-tab"
         and is NOT the Textual key — do not pass it to Binding).
       - Optional minimal `CSS`/`DEFAULT_CSS` giving `#transcript` `height: 1fr` and `#status`
         `dock: bottom` (discretionary layout).
       - `__init__(self, repl)`: store `self.repl = repl`; `self._stream_buf = ""`.
         Expose convenience: use `self.repl.harness`, `self.repl.state`, `self.repl.status`
         everywhere (single source of truth; `/clear` mutates `repl.state`).
       - `compose(self)`: yield `RichLog(id="transcript", markup=True, wrap=True, highlight=False)`,
         then `Static(id="stream")` (the in-flight streaming line — RESEARCH Pitfall 4 split-widget
         approach: stream tokens into this Static via `.update(buffer)` to avoid RichLog's
         newline-per-write behavior), then `Input(id="prompt", placeholder="agent86> ")`, then
         `StatusFooter(id="status")`.
       - `on_mount(self)`: set `self.query_one("#status", StatusFooter).status = self.repl.status`;
         write each `startup_notes(self.repl)` string into the RichLog (dim); focus the input.
    2. Input submission — `on_input_submitted(self, event: Input.Submitted)`:
       - `line = event.value.strip()`; clear the input (`event.input.value = ""`); if empty, return.
       - Echo the user line into the transcript (e.g. `[bold]> {line}[/bold]`).
       - `result = handle_command(self.repl, line)`. If `result.action == "exit"`: `self.exit()`.
         If `"turn"`: call `self._start_turn(line)`. Else (`"handled"`/`"noop"`): if `result.render`
         is not None, `RichLog.write(result.render)`; then set the footer to idle by reassigning
         `footer.status = self.repl.status` (so /mode and /model changes show immediately).
    3. Worker start — `@work(thread=True, exclusive=True) def _run_turn(self, line: str)`:
       body is exactly `run_turn_worker(self.repl.harness, line, self.repl.state, self.post_message)`.
       And `_start_turn(self, line)`: set `self.repl.status.working = True`,
       `self.repl.status.phase = "thinking"`, reassign `footer.status`, disable the input
       (`input.disabled = True`), reset `self._stream_buf = ""`, then call `self._run_turn(line)`.
    4. Message handlers (main thread; NEVER touch widgets from the worker — RESEARCH Anti-Patterns):
       - `on_turn_delta(self, m: TurnDelta)`: `self._stream_buf += m.text`;
         `self.query_one("#stream", Static).update(self._stream_buf)`; keep
         `status.working=True, status.phase="thinking"`, reassign footer; scroll transcript to end.
       - `on_tool_announce(self, m: ToolAnnounce)`: flush any `_stream_buf` into the RichLog as a
         finalized assistant entry and clear the Static + buffer; `RichLog.write(m.text.strip())`;
         set `status.phase = m.label` (e.g. "running write_file"), `status.working=True`, reassign footer.
       - `on_approval_request(self, m: ApprovalRequest)`: define
         `def _resolve(approved: bool | None) -> None: m.box["ok"] = bool(approved); m.event.set()`
         and call `self.push_screen(ApprovalModal(m.tool_name, m.preview), _resolve)`.
       - `on_turn_done(self, m: TurnDone)`: flush `_stream_buf` to the RichLog (prefixed
         `[bold cyan]agent86[/bold cyan] `) and clear the Static + buffer; `self.repl._refresh_status()`
         (sets working=False + idle stats); reassign footer; re-enable + refocus the input.
       - `on_turn_error(self, m: TurnError)`: flush buffer; write `[red]error:[/red] {m.error}` to the
         RichLog; `self.repl._refresh_status()`; reassign footer; re-enable + refocus the input.
       Use Textual message-handler naming: either `def on_turn_delta` (snake_case of the class) or the
       `@on(TurnDelta)` decorator — pick one and be consistent.
    5. `action_cycle_mode(self)`: `self.repl._cycle_approval()`; reassign `footer.status`.
    6. `def run_tui(cfg: Config, resume: str | None = None) -> None`: build `repl = _Repl(cfg, resume)`
       (reuses harness/state/status setup incl. resume), then `Agent86App(repl).run()`. Import `_Repl`
       inside `run_tui` (keep top-of-module clean of ui.repl if you prefer; either is fine since this
       whole module is lazy-loaded). Add `__all__ = ["Agent86App", "run_tui"]`.

    Notes: `post_message` is thread-safe (RESEARCH Pattern 1) so the worker calls `self.post_message`
    directly — do NOT use `call_from_thread`. Do NOT use `push_screen_wait` from the thread worker.
