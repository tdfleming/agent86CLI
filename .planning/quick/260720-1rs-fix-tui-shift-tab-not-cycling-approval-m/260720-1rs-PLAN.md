# Quick Task 260720-1rs: Fix TUI shift+tab not cycling approval mode

## Objective

In the TUI, pressing **Shift+Tab** does nothing — it should cycle the approval mode
(ask → auto → deny), as advertised by the footer's `[Shift+Tab]` hint.

Root cause: Textual's `App` ships a **default `shift+tab` binding for focus traversal**
(`focus_previous`). `Agent86App.BINDINGS` declares `Binding("shift+tab", "cycle_mode", ...)`
WITHOUT `priority=True`, so when the prompt `Input` is focused the focus-traversal handling
wins and `action_cycle_mode` never fires. This is the same "default binding intercepts the key"
footgun the 02-02 spike documented for `enter`, and the arrow/escape keys already solved with
`priority=True` + `SkipAction` fallthrough.

## Fix

1. Mark the shift+tab binding `priority=True` so the app hotkey is checked before focus
   traversal and before the focused widget's bindings (Textual docs: priority bindings are the
   documented mechanism for app-level hotkeys that must beat focus/tab handling).
2. Guard `action_cycle_mode` so it does NOT hijack Shift+Tab while a modal picker
   (`ModePickerModal` / `ModelPickerModal` / `ApprovalModal`) is on the screen stack — raise
   `textual.actions.SkipAction` when a modal is active, mirroring the existing
   `action_palette_up`/`_down`/`_dismiss` pattern, so Shift+Tab still navigates inside modals.

## Tasks

### Task 1: Make shift+tab a priority binding and guard it
- **files**: `src/agent86/tui/app.py`
- **action**:
  - Change `Binding("shift+tab", "cycle_mode", "cycle approval mode")` to include
    `priority=True` (keep the description so it still shows in the footer).
  - In `action_cycle_mode`, before cycling, raise `textual.actions.SkipAction()` when a modal
    is open. Detect "modal open" the same way the other palette actions detect their skip
    condition — reuse the existing helper/check in this file if one exists (e.g. checking
    `len(self.screen_stack) > 1` or `self.screen is not <base screen>`); match the pattern
    already used by `action_palette_dismiss`. Import `SkipAction` consistently with how the
    existing palette actions import/raise it.
- **verify**: `python -m pytest tests/tui -q` passes; the shift+tab binding entry has
  `priority=True`.
- **done**: Shift+Tab cycles approval mode from the main screen; a pushed modal still receives
  its own Shift+Tab.
